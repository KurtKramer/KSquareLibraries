45  /**
46  * @brief Constructs a date from a displayable string.
47  * @details Strings can be both all numeric or combination of alpha numeric. Each part of the date, (Year,
48  * Month, Day) can be separated by ' ', '/', '\\t', '\\n', or '\\r'. Examples of valid strings are
49  * "1997/02/25", "02/25/1997", "Feb 02 1997", "02/Feb/1997", "1997 February 02". The ordering of
50  * the numeric fields will be determined by context with one of three possible formats, mm/dd/yyyy,
51  * yyyy/mm/dd, and dd/mm/yyyy.
52  *
53  * A flaw in this schema is that if the day of the month is in the range of (1<->12), an all numeric
54  * format is used, and the ordering is supposed to be "dd/mm/yyyy" this routine will assume that it
55  * is looking at "mm/dd/yyyy" with the result of swapping the "Month" and "Day" fields. As a rule
56  * I try to always use the alpha numeric version of the month.
57  * @param[in] s String representing data that is to be parsed.
58  */
