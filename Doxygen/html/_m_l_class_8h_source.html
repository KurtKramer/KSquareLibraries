5  *@class KKMachineLearning::MLClass
6  *@brief Represents a "Class" in the Machine Learnining Sense.
7  *@author Kurt Kramer
8  *@details
9  *@code
10  **********************************************************************
11  ** MLClass *
12  ** *
13  ** Each instance of this class represents a single Class as used in *
14  ** Machine Learning sense. Each instance of 'FeatureVector' class *
15  ** will point to an instance of this class. There can only be one *
16  ** instance of each Class in memory. Specificacly the 'name' field *
17  ** will be unique. This is enforced by making the constructor and *
18  ** destructor private. The only way to create a new instance of a *
19  ** 'MLClass' object is to call one of the the static methods of *
20  ** 'CreateNewMLClass'. These methods will look for a instance *
21  ** of 'MLClass' that already exists. If one does not they will *
22  ** then create a new one. *
23  ** *
24  ** Please refer to MLClassList at bottom of file. That class is *
25  ** the object you will most be unsing when dealing with Images. *
26  ** classes. *
27  ** *
28  ** UmiClass *
29  ** There is a special relationship between this class and a class *
30  ** called 'UmiClass' in the 'UnManagedInterface' library'. UmiClass*
31  ** is a managed c++ version of this class. There is a one-to-one *
32  ** corespondance between the two classes. When ever a instance *
33  ** of 'UmiClass' gets created it will automatically call the the *
34  ** the static method 'MLClass::CreateNewMLClass' to get the *
35  ** unmanaged version of the class. *
36  **********************************************************************
37  *@endcode
38  *@see KKMachineLearning::MLClassList, KKMachineLearning::FeatureVector, UnManagedInterface::UmiClass
39  */
