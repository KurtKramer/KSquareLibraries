3  *@mainpage KSquareLibraries

5  *@section KSquare-Introduction Introduction to KSquare Libraries
6  * The libraries with their included functions/classes are the accumulation of work
7  *that I have built up over the last several years, mostly during my graduate program.
8  *This has become a useful tool-kit for several projects that I have worked on. The
9  *code is meant to be O/S neutral such that it can be compiled and used in either Windows
10  *or Linux.
11  *
12  *@section KKBase KKBase Basic Routines
13  *Functionality that is common to other libraries and applications. such as @link KKB::KKStr string management @endlink,
14  * @link KKB::Matrix Matrix @endlink operations, @link KKB::Raster Image Processing @endlink,
15  * @link KKB::Tokenizer Token Parsing @endlink, @link StatisticalFunctions.h Statistics @endlink,
16  * Histogramming, and common operating system routines. Most O/S specific code is implemented in the module
17  * @link OSservices.h "osServices.cpp" @endlink.
18  *
19  *@section KKLineScanner Line Scanner
20  *Routines to support Line-Scan camera imagery files. Classes and code that support
21  * the processing of data from line-scan cameras. Support for @link KKLSC::FlatFieldCorrection Dynamic-Flat-Field @endlink
22  *correction. Storage and retrieval of imagery data in 2,3,4, and 8 bit-depth
23  *gray-scale with support for embedding instrumentation data as well as text.
24  *
25  *@section KKMachineLearning Machine Learning.
26  *Classes and Code that support Machine-Learning implementations in the KKMLL namespace.
27  *Examples:
28  * - @link KKMLL::FeatureFileIO FeatureFileIO @endlink several common feature data file formats are support. (sparse, arff, c45, etc...)
29  * - @link KKMLL::TrainingConfiguration2 Training-Model-Configuration @endlink class and routines that maintain classifier parameters; such as classifier type.
30  * - Machine Learning Classes (@link KKMLL::MLClass MLClass @endlink ) and containers for tracking @link KKMLL::MLClassList lists of classes @endlink.
31  * - Hierarchical Class naming is supported.
32  * - Containers for Feature Data @link KKMLL::FeatureVector "FeatureVector" @endlink, @link KKMLL::FeatureVectorList "FeatureVectorList" @endlink, stratifying by class.
33  * - Feature data @link KKMLL::NormalizationParms normalization @endlink routines.
34  * - @link KKMLL::ConfusionMatrix2 Confusion-Matrix @endlink
35  * - @link KKMLL::CrossValidation CrossValidation @endlink - Example 10 fold CV; also (N x X) cross Validation; typically used by grading a classifier.
36  * - Classifier Implementations.
37  * -# Pair-Wise Feature selected SVM (Unique set of features for each pair of classes).
38  * -# Common Features SVM (One set of features for all class pairs).
39  * -# USF @link KKMLL::ModelUsfCasCor Cascading correlation @endlink neural networks (USF-Cas-Cor).
40  * -# @link KKMLL::ModelDual Dual Classifier @endlink, simple two level classifier that utilizes two classifiers to make a multi level prediction.
41  *
42  *@section KKJobManager Job Management
43  *Framework that assists in implementing batch processes on multiple cpu's across multiple nodes. Implemented under
44  * namespace KKJobManagment.
45  *
46  *@section OutsideLibrariesUsed Outside Libraries
47  *External libraries required for full functionality. That is libraries and code from other
48  *sources. Outside Libraries FFTW and ZLIB To use all the classes in this Library you will
49  *need the libraries fftw and lib123. "fftw" stands for "Fastest Fourier Transform in The West".
50  *It can be downloaded from http://www.fftw.org/. "zlib123" is a library that is used to compress
51  *and un-compress data. can be found at http://www.zlib.net/ and is used by the Compressor class.
52  *Their are two Macros that determine weather the two libraries listed above are utilized or more
53  *simple implementations included in KKBase. 'FFTW_AVAILABLE' - Indicates that you want to utilize
54  *the FFTW library. otherwise a simple Fourier transform will be utilized. 'ZLIB_AVAILABLE' -
55  *Indicates that 'zlib' library is available. If not defined the class Compressor will not do
56  *anything.
