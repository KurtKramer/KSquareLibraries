56  *@class TrainingProcess2 TrainingProcess2.h "KSquareLibraries\KKMachineLearning\TrainingProcess2.h"
57  *@brief Manages the creation and loading of training models.
58  *@details A trained model can either be built from scratch using specified training data or a persistent instance
59  * can be loaded from a XMLStream. There are several static methods that are best used to manage the various situations
60  * such as "CreateTrainingProcess", "CreateTrainingProcessForLevel", "CreateTrainingProcessFromTrainingExamples",
61  * "LoadExistingTrainingProcess". The result of any of these methods is a TrainigProcess2 instance that you
62  * use to create a Classifier instance.<p>
63  *
64  * Supporting Classes:
65  * - TrainingConfiguration2 Manages defines the parameters of a classifier such as:
66  * -# Type of algorithm
67  * -# List Classes of and where training examples can be loaded from.
68  * -# Sub-Classifiers that is for any given class another "TrainingConfiguration2" can be specified indicating another
69  * classifier to be used when that class is predicted.
70  * -# Class Weights
71  *
72  * - Model Base class for all algorithms. SupportVectorMachine(SVM), USFCasCor, BFS-SVM, etc ....
73  *
74  * - Classifier2 You construct an instance from a 'TrainingProcess2' instance; this is the class that manages predictions.
75  *
76  * Sub-Classifiers: For each sub-classifiers specified in the "TrainingConfiguration2" another instance of "TrainingProcess2"
77  * will be created. Data member "subTrainingProcesses" will keep a list of these "TrainingProcess2" instances.
78  */
